# Security, Governance, and Lineage

The platform is designed for responsible local review. It uses synthetic data only and avoids credentials, customer exports, production secrets, and live cloud calls.

## Final Governance Review

- Synthetic-only data policy: committed data is fictional NexaFlow data.
- No real PII: the repository must not store customer data or regulated personal information.
- Least privilege design: future Azure deployment should use scoped managed identities and workspace roles.
- Key Vault future pattern: secrets belong in Azure Key Vault, not files or CI variables committed to Git.
- Managed Identity future pattern: services should authenticate with managed identities rather than long-lived secrets.
- Purview future pattern: data assets, lineage, metric definitions, and semantic models should be registered.
- Azure Monitor future pattern: pipelines, jobs, refreshes, model drift, and data quality should emit monitorable signals.
- Power BI RLS future pattern: row-level security should be reviewed against tenant, region, and ownership requirements.
- No live deployment by default: this repository is local-first and Azure-mappable.
- Reproducible evidence: milestone evidence is generated by deterministic scripts.
- Deterministic CI: CI performs quality and evidence checks only.
- Model-risk limitations: churn, segmentation, and recommendations are synthetic demonstrations.
- GenAI guardrails: default insight generation is deterministic and evidence-grounded.
- Human review: production use requires human approval and governance review.

## Lineage Pattern

The final lineage is:

```text
synthetic data -> raw landing -> ingestion validation -> trusted accepted data
-> analytics, ML and experiments -> product insights -> Power BI-ready reporting
-> portfolio evidence
```
